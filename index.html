UX Design or User Experience Design, is a practice that describes the overall design of a product. It encompasses branding, user research and persona creation, wireframing, user paths and aesthetics. A UX designer must consider how the identified typical user will want to use the product or software and then design the mechanisms that will allow them to do so. For this to happen they must understand the nature of the intended client(s) and then make educated design decisions based on user testing and feedback as well as existing design patterns. From here, the designer will then develop “User Journeys” which will describe the overall collection of use cases for the product. These use cases will depict step by step how a user or client will progress through or use the product as they try to reach their goal. There are a number of existing theories and patterns that describe parts of this process such as “Progressive Disclosure”.

Progressive Disclosure is a design pattern that is used to simplify the users path or journey through an interactive software or menu system. The idea is to remove clutter and overcrowding of choice and to only present the relevant options to the user. The most prominent features of the program will usually be presented earlier in the menu system, with more granular or rarely used functions or options being presented in more specific areas further along the journey.

A good example of this is in Windows based programs. Typically we are presented with a menu in the top left of our screen. The first menu option is usually “file” and it contains sub options like “new”, “open”, “save”. Each of these options typically present the user with further options in the form of a popup window. Each of those sub options are related specifically to the original choice. Eg you would not find the option to open a new document inside a “save” popup dialogue window. The user is guided safely through a path or journey intended to achieve a specific result.

Studies have shown that Progressive Disclosure not only makes a product much more logically accessible, but also allows for the user to learn how to use the product much more quickly, while also encouraging exploration of more hidden features.
